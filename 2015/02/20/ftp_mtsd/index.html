      虚拟主机FTP上传的MTSD连接超时问题

      Post on 2月 20 2015

      问题描述

      这里给出Mac OS X下的FileZilla的Log：

      状态: 正在解析 desgemini.co 的地址
      状态: 正在连接 157.7.65.33:21…
      状态: 连接建立，等待欢迎消息…
      状态: 初始化 TLS 中…
      状态: 正在验证证书…
      状态: TLS connection established.
      状态: 已连接
      状态: 读取目录列表…
      命令: PWD
      响应: 257 当前目录是”/“
      命令: TYPE I
      响应: 200 类型设置为I
      命令: PASV
      响应: 227 Entering Passive Mode (157,7,65,33,137,66).
      命令: MLSD
      错误: 无法建立数据连接：ETIMEDOUT - 连接尝试超时

      横向测试

      分别测试了如下FTP软件：

      应用软件 实测结果
      FileZilla 3.8 正常
      FileZilla 3.10.1.1 MLSD时超时
      CyberDuck 无法获取目录，估计是MLSD时超时
      YummyFTP 无法获取目录，估计是MLSD时超时

      Google it!

      Google得到以下几个有切题干货的网址：

      1. https://www.flashfxp.com/forum/flashfxp/frequently-asked-questions-faq-/13575-list-list-al-mlsd-stat.html
      2. http://serverfault.com/questions/224510/unable-to-connect-to-ftp-connection-timeout-after-mlsd
      3. https://trac.cyberduck.io/wiki/help/en/howto/ftp

      归纳一下

      1. LIST，LIST -al，MLSD，STAT -L几种FTP命令的异同
        1. LIST：最普通的列出ftp服务器上的目录，类比ls
        2. LIST -al：LIST命令带参数，类比ls -al
        3. MLSD：比较新的FTP命令，提供了更标准，更详细，机器可读性更强的目录信息。
        4. LIST -L：类似ls -l
      2. MLSD连接超时的原因有很多种，有可能是系统防火墙阻拦，也有可能是路由器的问题，还有可能是主动被动的问题，需要尝试各种可能才能确定。我没能找到解决的方法，因此退而求其次，寻求禁止MLSD，改用LIST的方法。
      3. Yummy FTP和FileZilla均没有提供禁止MLSD改用LIST的选项，但stackoverflow上喷FileZilla远远比Yummy FTP，这大概是因为开源项目自带使命光环吧。
      4. CyberDuck将获取ftp目录的设置选项隐藏在Mac OS X user defaults system。需要在terminal中设置，设置命令如下：
        1. STAT的开启与关闭：defaults write ch.sudo.cyberduck ftp.command.stat false/true
        2. LIST -a的开启与关闭：defaults write ch.sudo.cyberduck ftp.command.lista false
        3. MLSD的开启与关闭：defaults write ch.sudo.cyberduck ftp.command.mlsd false

      设置完成后需重启CyberDuck。
      在禁用了MLSD并开启了LIST后，ftp功能就能正常使用了。
